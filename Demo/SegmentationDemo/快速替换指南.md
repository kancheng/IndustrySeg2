# 快速替换模型和测试数据指南

## 🚀 最快方法（推荐）

### 使用命令行参数

```bash
# 方式 1: 使用 dotnet CLI
dotnet run -- "你的模型路径.onnx" "你的图片路径.jpg"

# 方式 2: 直接运行可执行文件
SegmentationDemo.exe "你的模型路径.onnx" "你的图片路径.jpg"

# 示例
dotnet run -- "C:\Models\my-model.onnx" "C:\Images\test.jpg"
```

**优点**：无需修改代码，最灵活！

---

## 📁 简单方法：直接替换文件

### 替换模型
1. 将你的模型文件放到：`test/assets/Models/`
2. 重命名为：`yolov11s-seg.onnx` 或 `yolov8s-seg.onnx`

### 替换图片
1. 将你的图片文件放到：`test/assets/Media/`
2. 重命名为：`people.jpg` 或 `street.jpg`

---

## 🔧 修改代码方法

### 修改模型路径
编辑 `Program.cs` 的 `GetModelPath()` 方法（约第 234 行）：

```csharp
// 修改这一行
var customModel = Path.Combine(projectRoot, "test", "assets", "Models", "你的模型名称.onnx");
```

### 修改图片路径
编辑 `Program.cs` 的 `GetImagePath()` 方法（约第 260 行）：

```csharp
// 修改这一行
var customImage = Path.Combine(projectRoot, "test", "assets", "Media", "你的图片名称.jpg");
```

## ✅ 验证替换是否成功

运行程序后，查看控制台输出：

```
使用模型: [你的模型路径]
使用圖片: [你的图片路径]
Loaded ONNX Model: Segmentation (yolo v11)

Inference Results: X objects
```

如果看到以上信息，说明替换成功！

---

## ⚠️ 注意事项

1. **模型要求**：
   - 必须是 ONNX 格式的分割模型
   - 支持 YOLOv5-v12 系列

2. **图片要求**：
   - 支持常见格式：`.jpg`, `.png`, `.bmp`
   - 无尺寸限制（会自动调整）

3. **路径格式**：
   - Windows: `C:\路径\文件.onnx`
   - 相对路径: `test/assets/Models/模型.onnx`
